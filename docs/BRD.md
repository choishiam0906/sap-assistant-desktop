# BRD (Business Requirements Document)
# SAP 운영 자동화 AI 봇

## 1. 비즈니스 목적

### 프로젝트 배경
SAP 운영 노하우를 AI 봇으로 자동화하여 운영 효율성을 높인다.

> **v2.5 Desktop 클라이언트 구현 완료 (2026-03-06)**
> Electron 기반 Desktop 앱으로 채팅 UI, CBO 소스 분석, 설정 관리까지 전체 Renderer UI 구현 완료.

### 비즈니스 가치
1. **운영 효율성 향상**: T-code 검색/절차 확인 시간 50% 절감
2. **지식 보존**: 개인 종속 지식을 조직 자산으로 전환
3. **온보딩 가속**: 신규 인력 학습 기간 단축 (3-6개월 → 1-2개월)
4. **오류 대응 속도**: 적절한 디버깅 도구 즉시 안내
5. **인프라 비용 절감**: Desktop 로컬 실행으로 서버 인프라 의존도 감소

## 3. 비즈니스 요구사항

### BR-1: 자연어 기반 SAP 운영 질의응답
- **설명**: 사용자가 한국어 자연어로 SAP 운영 관련 질문을 하면, AI가 적절한 답변을 제공
- **비즈니스 규칙**: 답변에는 항상 관련 T-code, 실행 절차, 주의사항이 포함되어야 함
- **수용 기준**: 기존 PPT에 있는 내용에 대해 80% 이상 정확한 답변

### BR-2: 사용자 인터페이스
- **설명**: Desktop 앱에서 직관적인 채팅 UI로 SAP 운영 질의응답을 수행
- **현재 상태**: Electron Desktop 채팅 UI 구현 완료 (React 18 + Zustand)
- **향후 계획**: Microsoft Teams 통합 (Copilot Studio를 통한 기업 표준 배포) 검토 예정
- **수용 기준**: Desktop 앱에서 3클릭 이내 첫 질문 전송 가능

### BR-3: 지식 베이스 관리
- **설명**: 관리자가 SAP 운영 지식을 추가/수정/삭제할 수 있는 관리 도구
- **비즈니스 규칙**: 승인된 관리자만 접근 가능
- **수용 기준**: 지식 추가 후 5분 이내 챗봇 응답에 반영

### BR-4: 사용 현황 모니터링
- **설명**: 봇 사용 현황, 자주 묻는 질문, 미답변 질의를 파악할 수 있는 대시보드
- **수용 기준**: 일별/주별 사용 통계, 인기 질문 Top 10 제공

## 4. 비즈니스 제약사항

| 제약사항 | 설명 |
|---------|------|
| 보안 | Desktop: API Key를 keytar SecureStore에 OS 키체인으로 암호화 저장 |
| 데이터 | SAP 시스템 자격증명은 봇에 저장하지 않음 |
| 비용 | LLM API 비용 월 예산 내 운영 (Desktop 로컬 실행으로 서버 비용 절감) |
| 규정 | 개인정보(사용자ID 등) 처리 시 사내 보안 정책 준수 |
| Electron 격리 | contextBridge 기반 Renderer ↔ Main 프로세스 격리 |

## 5. ROI 분석 (추정)

### 비용 절감 효과
- SAP 운영자 1인 기준 T-code 검색/절차 확인: 일 30분 → 15분 (50% 절감)
- 운영자 10명 기준: 월 약 **50시간** 절감
- 신규 인력 온보딩: 건당 약 **80시간** 절감 (연 2-3명)
- Desktop 버전으로 서버 인프라 비용 절감: 별도 채팅 서버 불필요

### 투자 비용
- 초기 개발: 4주 (개발 인력 1명)
- LLM API: 사용량 기반 과금 (사용자 API Key)
- 인프라: Knowledge API 서버만 운영 (Desktop은 로컬)

### 예상 ROI
- 손익분기점: 런칭 후 약 2개월
